Formは単純な文字列のみを受け取ることになる  
リクエストに含めるかどうかをクライアント側で制御する（プロフィールで変更していない長い文字列をいちいち送信しないように等）場合、かつすべてのパラメータが文字列ならばFormでよい

対して、オブジェクト等の複雑な構成のパラメータを渡す必要があったり、Nullableな値を渡す必要があるとかの場合はJSONを使うことになる

基本的にはJSONの方があらゆる面で優れているのでそっちに統一してもいい  
どうせPOST等の通信もAjax経由でしか行わないので

一応、Formでも数値（整数型）はちゃんとパース出来るのを確認している  
小数とかは知らない　booleanは確実にちゃんと動作しないのでJSONにしないといけない（そもそもチェックボックスはboolean型で渡されない）

APIなど、ブラウザ以外の場所から叩くことを想定しているものはすべてJSONにする  
なんなら全部fetchAPI経由なんだから全部JSONでもいいくらいだけど、一応ログインとかは普通のフォームに入力してもらってリクエスト経由で進めていく想定でいるのでFormということになっている
