# ポータル
root (GET) -> ログイン済みなら /profile へ
- /entry
  - (GET): 玄関
  - (POST): ログイン
  - (DELETE): ログアウト
  - register (POST): 新規登録
- /profile
	- (GET): 編集・設定画面
	- (PATCH): 更新処理
	- (DELETE): 削除
- /user
	- (GET): リスト
	- (POST): 絞り込み検索API
	- /:id (GET): 個別プロフィール閲覧
- /auth
  - (GET): ログイン状態なら一時キーを発行
  - (POST): 一時キーを受け取って認証し、user.idを返す
- /info
- /rule
- /report

## ポータルを利用した新規登録・ログイン処理
まず先にポータルにアカウントを作成しておく（通常の流れ）

### 1. 入力するパターン
1. アプリ側で、ポータルアカウントIDとパスワード、および個別入力情報を入力する
2. 登録時にポータル側に問い合わせてIDとパスワードの照合をする
3. 正しい組み合わせなら登録成功、間違っているならエラー（新規作成に誘導する）
### 2. ログイン情報を利用するパターン
1. cookie-sessionに保持しているポータルログイン情報を登録時アプリ側で取得する
2. ポータル側に問い合わせてログイン状態が正しいか確認する
3. 問題なければ登録成功、間違っていたりそもそもログインしてなかったならエラー
### 3. ちゃんとする
1. アプリ側からwindow.openでポップアップウィンドウにログインページを開く
2. ログイン状態ならログイン一時キーが発行されるのでそれを返す
3. 一時キーをアプリ側のバックエンドに送信
4. アプリとポータルのバックエンド同士で通信して一時キーを認証する
5. 問題なければログイン状態をアプリ側が受け取る
